## Release v1.6.0

**Release Date:** June 14, 2026

Релиз **v1.6.0** относительно **v1.5.2**: серия блокирующих багфиксов (SS-клиент, порт vless+hy2, TG-бэкап, перезапуск панели, flow VLESS, групповое назначение инбаундов, загрузка геофайлов), UI-правки и новые возможности (Clash YAML, импорт outbound из ссылки, поле TCP MSS).

**Миграция БД:** нет.

---

### Bug Fixes (P0 — блокеры)

- **#86 — Ошибка создания SS-клиента** (`web/service/client.go`, `xray/api.go`)
  - При создании клиента с Shadowsocks-инбаундом панель показывала ошибку, хотя клиент успешно создавался в БД.
  - Исправлен ключ payload для Xray gRPC: `"method"` → `"cipher"`. Добавлены nil-safe type assertions в `xray/api.go` для предотвращения паники.

- **Standalone: VLESS + Hysteria2 на одном порту** (`web/service/inbound.go`)
  - В standalone-режиме `checkPortExist` блокировал конфигурации VLESS (TCP) + Hysteria2 (UDP) на одном порту.
  - Логика теперь разрешает совместное использование порта разными транспортными уровнями — аналогично multi-node.

- **TG-бэкап не запускался периодически** (`web/web.go`, `web/job/stats_notify_job.go`)
  - Cron-задача регистрировалась только при включённом боте на старте; изменение настроек без перезапуска панели не давало эффекта.
  - `StatsNotifyJob` зависела от `IsXrayRunning()` — бэкапы пропускались при остановленном Xray.
  - Задачи теперь регистрируются безусловно и проверяют настройки в runtime.

- **Групповое назначение инбаундов — только добавление** (`web/service/client.go`, `web/controller/client_group.go`, `panel/components/GroupsPage.tsx`)
  - Операция всегда выполняла объединение (add), уменьшить набор инбаундов было невозможно.
  - Добавлен параметр `mode: "add" | "replace"` (default `"replace"`). UI отправляет `"replace"`, заменяя всё.
  - Обновлены подсказки в локализационных файлах.

- **#90 — Загрузка геофайлов падает для файлов >32 МиБ** (`web/web.go`, `web/service/server.go`, `node/api/server.go`)
  - Gin по умолчанию ограничивал multipart-память в 32 МиБ; файлы (например, ru geoip.dat ~50 МиБ) отклонялись раньше сервисной проверки.
  - Лимиты подняты до 256 МиБ во всех трёх слоях.

- **VLESS xtls-rprx-vision: невозможно вернуть flow в «Нет»** (`web/service/inbound.go`, `web/service/vless_flow_compat.go`)
  - Sanitize-функция сканировала всех клиентов инбаунда и восстанавливала stale flow при наличии у любого из них непустого значения.
  - Теперь `clients[0].flow` является единственным авторитетным источником; значение явно распространяется на всех остальных клиентов.

- **«Перезапустить панель» перезапускал только Xray-ядро** (`web/controller/setting.go`)
  - Кнопка вызывала `RestartPanel` (SIGHUP = мягкий рестарт HTTP-сервера). Заменена на `RestartContainer` (SIGTERM к PID 1 + `os.Exit(0)` fallback) для полного рестарта процесса.

### UI Fixes (P1)

- **#87 — Кнопка «Случайный SNI» накапливала hex-метки** (`panel/lib/inboundDefaults.ts`)
  - Каждый клик добавлял новый 6-символьный hex-префикс к уже существующему. Теперь все предыдущие `[0-9a-f]{6}.` метки удаляются перед добавлением нового.

- **TCP Fast Open ломал layout формы Inbound** (`panel/components/InboundsPage.tsx`)
  - Чекбокс в `grid-cols-2` рядом с `Input`/`Select` создавал разные высоты строк. Чекбоксы sockopt вынесены в отдельный flex-ряд.

- **#89 — WireGuard QR включал весь текст вместо `.conf`** (`panel/components/ClientsPage.tsx`, `panel/components/sub/SubPageRenderer.tsx`)
  - QR-код содержал заголовки и пояснительный текст, что приводило к ошибкам при импорте в WireGuard-приложения.
  - В QR теперь передаётся только блок `[Interface]...[Peer]`.

### New Features (P2)

- **#88 — Clash YAML (`?format=clash`)** (`sub/subController.go`, `sub/clash_yaml.go`)
  - Добавлена поддержка query-параметра `?format=clash` (и алиасы `clash-meta`, `mihomo`) для получения подписки в формате Clash YAML.
  - Поддерживаются протоколы: vmess, vless, trojan, shadowsocks, hysteria2.
  - Ответ возвращается как `text/yaml; charset=utf-8` без base64.

- **Импорт outbound из ссылки** (`panel/components/xray/OutboundsBuilder.tsx`, `panel/lib/parseProxyLink.ts`)
  - В конструкторе Outbounds появилась кнопка «Import from link».
  - Поддерживаются `vless://`, `vmess://`, `trojan://`, `ss://` (один или несколько, по одному на строку).
  - Результат добавляется как новые строки в текущий список outbounds.

- **Поле TCP MSS в форме инбаунда** (`panel/lib/inboundDefaults.ts`, `panel/components/InboundsPage.tsx`)
  - Добавлено числовое поле **TCP MSS** в секцию Sockopt формы инбаунда.
  - 0 = не задано (поле не попадает в конфиг); Xray читает `sockopt.tcpMss` самостоятельно.

- **Модалка редактирования группы — как у клиента** (`panel/components/GroupsPage.tsx`, `panel/public/locales/en.json`, `panel/public/locales/ru.json`)
  - Двухколоночная вёрстка: слева «Идентификаторы» (имя/описание) и «Данные группы» (счётчик клиентов, создание/обновление), справа — «Срок действия», «Трафик», «Инбаунды», «Настройки HWID», «IP Limit». Те же иконки и стиль, что и в модалке клиента.
  - Одна общая кнопка **Save** внизу справа: применяет только тронутые секции к bulk-эндпойнтам (`group/update`, `bulk/setExpiry`, `bulk/setTrafficLimit`, `bulk/setHwidLimit`, `bulk/setIPLimit`, `bulk/assignInbounds`).
  - Опасные действия (включить/отключить всех, сбросить трафик, очистить HWID, удалить всех клиентов) — иконками в правом верхнем углу шапки модалки.
  - Добавлены недостающие переводы (`pages.groups.groupName`, `sectionIdentity`, `sectionGroupData`, `expiryDate`, `trafficLimitGB`, …) в `en.json` и `ru.json`.

### Operational Notes

- Обновляются **`sharx`** и **`sharxnode`** (изменения в `node/api/server.go`).
- Миграций БД нет.
- После обновления проверьте:
  - Создание SS-клиента — без ошибки.
  - Загрузка геофайлов >32 МиБ.
  - Перезапуск панели через интерфейс — полный рестарт процесса.
  - Групповое переназначение инбаундов — «replace» работает.

4. Проверьте:
