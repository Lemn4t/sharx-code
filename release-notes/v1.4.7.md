## Release v1.4.7

**Release Date:** June 1, 2026

Релиз **v1.4.7** относительно **v1.4.6**: feature-релиз с переключением версии Telemt (как у Xray), поддержкой клиента **INCY** на странице подписки и исправлением сохранения фильтров/колонок таблицы клиентов.

**Миграция БД:** `0045_node_telemt_version.sql` — колонка `telemt_version` в `nodes` для кэша версии Telemt с worker.

---

### Bug Fixes

- **Сброс фильтров и колонок на странице Clients после refresh** (`panel/components/ClientsPage.tsx`, `web/service/setting.go`, `web/controller/setting.go`)
  - **Причина:** save prefs выполнялся до завершения load — дефолты перезаписывали сохранённое состояние при каждом обновлении страницы.
  - **Фикс:** load → `tablePrefsReadyRef` → debounced save; ключ `clientsTablePrefs:user:{id}` per-user в БД с миграцией legacy localStorage-ключа.

### Features & Improvements

- **Переключение версии Telemt с дашборда** (`telemt/install`, `web/controller/server.go`, `web/service/server.go`, `node/api/server.go`, `node/telemt/manager.go`, `panel/components/DashboardPage.tsx`, `panel/components/NodesPage.tsx`)
  - **Dashboard → «Версия ядра»:** модалка с вкладками **Xray | Telemt**; список релизов с [github.com/telemt/telemt/releases](https://github.com/telemt/telemt/releases) (musl-бинарники, SHA256 verify).
  - **Standalone:** stop sidecars → замена `bin/telemt` → restart Telemt (Xray не трогается).
  - **Multi-node:** установка на **все** ноды через `POST installTelemtOnNodes/{version}`; локальный Telemt на панели не используется.
  - **Worker API:** `POST /api/v1/install-telemt/{version}` — stop → install → restart из последнего Apply snapshot.
  - **Статус:** `telemt.version` на дашборде (standalone); `telemtVersion` в worker `/status` и `/health`; колонка **Telemt Version** на странице **Nodes** (кэш из health-check).

- **INCY в списке приложений на странице подписки** (`panel/lib/sharxSubpageConfig.ts`, `sub/subController.go`, `sub/ua_dispatch.go`, `web/service/sharx_subpage_config.go`)
  - Deep link `incy://import/{url}` в каталоге приложений subpage builder.
  - Rollout в существующие блоки «Add to app» / installation guide.
  - User-Agent `incy` / header `x-client: incy` + HWID — в allowlist при включённом шифровании подписки (аналог Happ / v2raytun).
  - Routing profiles: preset `incy://routing/add/…` (как happ/sharx).

### Operational Notes

- Обновление **панели (`sharx`) и worker (`sharxnode`)** рекомендуется: установка Telemt на нодах использует новый endpoint worker'а.
  - Только панель — достаточно для INCY, clients prefs и Telemt в standalone.
  - Multi-node Telemt version switch — **обязательно** обновить worker до **1.4.7** на всех нодах (или сразу после смены версии с дашборда).
- **Миграция 0045** применяется автоматически при старте панели.
- Telemt install скачивает **официальные prebuilt** (linux amd64/arm64 musl); Rust/cargo build не используется.
- После смены версии Telemt sidecars перезапускаются автоматически, если были запущены до обновления.

### 📦 Upgrade from v1.4.6

1. Бэкап `.env` и БД.
2. Обновите образы **`sharx`** и **`sharxnode`** до **`1.4.7`** / **`latest`**.
3. Дождитесь миграции `0045` (лог старта / `SELECT telemt_version FROM nodes LIMIT 1;`).
4. Проверьте:
   - **Dashboard → Версия ядра → Telemt:** список версий загружается; после install — версия в статусе (standalone) или в колонке **Telemt Version** на **Nodes** (multi-node).
   - **Clients:** фильтры и видимые колонки сохраняются после F5 (per-user).
   - **Subscription page:** INCY в блоке «Add to app» / installation guide; deep link `incy://import/…` открывается в клиенте.
   - Routing profile с preset **Incy** — deep link `incy://routing/add/…`.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
