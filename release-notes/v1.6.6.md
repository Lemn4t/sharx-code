## Release v1.6.6

**Release Date:** June 15, 2026

Патч **v1.6.6** относительно **v1.6.5**: исправления Shadowsocks inbound — корректные ключи для AEAD 2022, сохранение метода в UI, синхронизация client secrets при смене cipher.

**Миграция БД:** нет.

---

### Bug Fixes

- **Shadowsocks inbound (метод / ключи)** (`panel/lib/shadowsocksKeys.ts`, `panel/lib/inboundDefaults.ts`, `web/service/shadowsocks_method.go`, `web/service/inbound.go`, `web/service/client.go`) — [#85](https://github.com/konstpic/SharX/issues/85)
  - **Симптом 1:** Xray не стартует с `proxy/shadowsocks_2022: … bad key` — server key одинаковой длины для всех методов.
  - **Симптом 2:** после сохранения инбаунда с методом ≠ `aes-256-gcm` при reopen метод откатывается на `aes-256-gcm` и появляется новый пароль.
  - **Симптом 3:** при смене метода у существующих клиентов остаётся `method` → `shadowsocks 2022 (multi-user): users must have empty method`.
  - **Симптом 4:** после исправления server key клиентские ключи в конфиге не обновлялись.
  - **Фикс:** для **2022** server/user key — base64 16/32 байт; панель читает `method`/`password` из корня `settings`; у клиентов 2022 в Xray JSON не задаётся `method`; при смене cipher пересоздаются client secrets и пересобирается inbound.

### Operational Notes

- Обновляется **панель (`sharx`)** — UI и backend; **`sharxnode`** без изменений, но обновление до той же линии версий рекомендуется.
- После деплоя panel: пересборка Docker-образа (`docker compose build sharx && docker compose up -d --force-recreate sharx`).
- Существующие SS-2022 инбаунды с невалидным server key будут автоматически получать новый ключ при следующем save/push; клиентам потребуется обновить subscription/ключи.
