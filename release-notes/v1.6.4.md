## Release v1.6.4

**Release Date:** June 18, 2026

Патч **v1.6.4** относительно **v1.6.3**: NAT для AmneziaWG sidecar, routing deep links Happ, короткие имена `.conf` по remark инбаунда.

**Миграция БД:** нет.

---

### Bug Fixes

- **AmneziaWG NAT не применялся** (`node/amneziawg/manager.go`) — [#14](https://github.com/konstpic/sharx-code/pull/14) от **[@Lemn4t](https://github.com/Lemn4t)**
  - `iptablesEnsureAppend` собирал `iptables -A -t nat …` вместо `iptables -t nat -A …`; правило MASQUERADE в таблице `nat` не создавалось, трафик через туннель мог не уходить в интернет.
  - Дополнительно: та же ошибка порядка аргументов в `iptablesHasRule` (`-C`), общий хелпер `buildIptablesArgs`, unit-тесты.

- **Длинное имя файла `.conf` на подписке** (`sub/wireguard_panel.go`, `sub/amneziawg_panel.go`, `panel/lib/wireguardConf.ts`, `panel/components/ClientsPage.tsx`, `InstallationGuideBlock.tsx`)
  - В текст подписки добавлена строка `Inbound: {remark}`; Download / AmneziaVPN используют remark инбаунда (`DE-Frankfurt.conf`), а не boilerplate из заголовка `AmneziaWG (UDP) — use the .conf block below…`.
  - **Clients → ключи:** скачивание `.conf` тоже по remark инбаунда.

### Features

- **Happ routing `onadd`** (`panel/lib/happRouting.ts`, `RoutingProfilesEditor.tsx`) — [#15](https://github.com/konstpic/sharx-code/pull/15) от **[@Lemn4t](https://github.com/Lemn4t)**
  - Пресет `happ_onadd` и генерация/парсинг `happ://routing/onadd/…` рядом с существующим `happ://routing/add/…`.
  - `incy://routing/add/…` и `sharx://routing/add/…` без изменений.

### Operational Notes

- Обновляются **панель (`sharx`)** и **`sharxnode`** (iptables fix в sidecar).
- После обновления panel: пересборка образа для UI подписки (`docker compose build sharx && docker compose up -d --force-recreate sharx`).
- AmneziaWG: в логах ноды ожидается `amneziawg routing OK` без warning `iptables append` на MASQUERADE.

1. Бэкап `.env` и БД (миграций нет).

```yaml

  sharx:
    image: {{SHARX_TAG_LATEST}}

      - NET_ADMIN
```
