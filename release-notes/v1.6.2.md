## Release v1.6.2

**Release Date:** June 15, 2026

Патч **v1.6.2** относительно **v1.6.1**: подписка WireGuard (AmneziaWG), безопасность mixed inbound, sing-box JSON, INCY rollout, чекбоксы в модалке инбаундов, тема Vision (Liquid Glass).

**Миграция БД:** **`0047_node_amneziawg_state.sql`** — колонка **`amneziawg_state`** в **`nodes`**.

---

### Bug Fixes

- **WireGuard / AmneziaWG в GetSubs** — протоколы `wireguard` и `amneziawg` включены в `allowedSub`; WG/AWG `.conf` не режется по `\n` в подписке.
- **AmneziaWG на странице подписки** — приложение `amneziawg` в каталоге; QR/copy wg-quick `.conf` с обфускацией Jc/H/S.
- **INCY нельзя было удалить** — убран принудительный rollout `SUBSCRIPTION_APP_ROLLOUT: ["incy"]`; INCY остаётся в каталоге, но не пере-добавляется при сохранении конфига подписки.
- **Чекбоксы в модалке инбаундов** — `preventLabelFocusScroll` в `CheckboxOptionCard`; TLS/accept proxy/xHTTP/mTLS чекбоксы вынесены из двухколоночных grid (`InboundsPage.tsx`, `checkbox-option.tsx`).
- **Mixed inbound без авторизации** — skeleton и rebuild settings всегда `auth: "password"` (пустой `accounts` при отсутствии клиентов вместо `noauth`) (`InboundsPage.tsx`, `web/service/inbound.go`).
- **AmneziaWG inbound UI** — transport step без VLESS flow / TCP / REALITY; sidecar fields нормализуются на save (`streamSettings: {}`).
- **sing-box «decode config»** — deep link использует `{urlJsonEncoded}`; кнопка скрывается, если JSON endpoint выключен (`sharxSubpageConfig.ts`, `AddToAppBlock`, `InstallationGuideBlock`).

### Features

- **AmneziaWG sidecar (amneziawg-go)** — отдельный протокол `amneziawg` (не Xray WG): инбаунд в панели, preview `.conf`, **`apply-sidecars`** / локальный Apply на panel/node, подписка с client `.conf`. Бинарники **`amneziawg-go`** + **`awg`** собираются в Docker (`DockerInit.sh`, нужны **git/make** в builder).
- **Nodes → Sidecars** — stop/restart AmneziaWG на воркере (как Telemt), поле **`amneziawg_state`**.
- **Vision (Liquid Glass)** — светлая тема с blur для панели (`panelTheme.ts`, `globals.css`) и страницы подписки (`subPageColorPreset.ts`, `subscription-shell.module.css`). В подписке: Theme → Light + Color palette → Vision.

### Operational Notes

- Обновляются **панель (`sharx`)** и **`sharxnode`** (AmneziaWG sidecar API + бинарники в образе).
- Применить миграцию **`0047`** после обновления панели.
- **AmneziaWG / WireGuard в Docker:** userspace **`amneziawg-go`** поднимает TUN. В compose для **`sharx`** (standalone sidecar) и **`sharxnode`** добавьте **`NET_ADMIN`** и проброс TUN с хоста, иначе в логах будет `CreateTUN … /dev/net/tun does not exist`:

```yaml
services:
  sharx:          # или sharxnode / node:
    cap_add:
      - NET_ADMIN
    devices:
      - /dev/net/tun:/dev/net/tun
```

  Образы также вызывают `scripts/ensure-dev-net-tun.sh` при старте, но **mount `/dev/net/tun` надёжнее**, чем один только `mknod` в контейнере. После правки compose: `docker compose up -d --force-recreate`.

- Sidecar-синк: **`POST /apply-sidecars`** на воркере и **`SyncWorkerSidecarsAsync`** в панели — peers AWG/Telemt обновляются **без рестарта Xray** (полный `apply-config` только когда меняется Xray JSON).
- Inbound **`amneziawg`**: в UI нет Xray transport (TCP/TLS/REALITY/flow); при save в БД пишутся пустые `streamSettings` / sniffing.
- Для sing-box: Settings → включить **JSON subscription endpoint** (`subJsonEnable`).
- WireGuard: проверить QR AmneziaWG на `/panel/sub/...`.
