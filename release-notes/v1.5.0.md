## Release v1.5.0

**Release Date:** June 4, 2026

Релиз **v1.5.0** относительно **v1.4.9**: тема панели **Star Wars** (киношный фон: звёзды, гиперпространство, лазеры), UX клиентов (дата истечения → **00:00**), правки редактора Xray JSON в Inbounds, надёжный старт **Xray/Telemt** на воркерах после обновления.

**Миграция БД:** нет.

---

### Features & Improvements

- **Тема панели Star Wars** (`panel/lib/panelTheme.ts`, `panel/app/globals.css`, `panel/styles/star-wars-theme.css`, `panel/app/layout.tsx`, `panel/components/panel/PanelShell.tsx`, i18n)
  - Пресет в настройках панели; шрифты **Orbitron** / **Pathway Gothic One**, палитра `#FFE81F`, звёздное поле, streaks гиперпространства, лазерные блики.
  - Киношные эффекты только в **оболочке панели** (`panel-cinema-bg`), не на странице подписки (`panel/lib/subPageColorPreset.ts` — только цвета).

- **Клиенты: дата без времени → полночь** (`panel/lib/format.ts`, `panel/components/ClientsPage.tsx`)
  - При выборе даты в `datetime-local` для срока действия подставляется **00:00** (как в v1.4.9), без сдвига на «сейчас».

- **Multi-node: устойчивый старт Xray и Telemt после рестарта/обновления** (`node/configpull/pull.go`, `node/config/config.go`, `node/main.go`, `web/service/node.go`)
  - **Воркер:** повторы `pull-xray-config` при старте (backoff), сброс устаревшего `nodeId` при HTTP 401 и повтор по `nodeAddress`, фоновые повторы раз в 60 с, пока Xray не запущен.
  - **Панель:** после успешного health check, если нода **online**, но Xray не крутится (или нужен Telemt, а он не активен) — автоматический полный **`apply-config`** (throttle 3 мин на ноду).

### Bug Fixes

- **Inbounds: Monaco «бесконечные» числа и Invalid JSON** (`panel/lib/inboundJsonMonacoSchemas.ts`, `panel/components/InboundsPage.tsx`, `panel/components/inbounds/InboundXrayCoreEditor.tsx`)
  - Убраны `min`/`max` у `port` в JSON Schema (Monaco перечислял 1–65535).
  - «Обновить из формы» не обнуляет редактор: `coreConfigPreviewNonce` + не рефетчить preview при dirty draft.

- **Star Wars: эффекты не на подписке** (`panel/styles/star-wars-theme.css`, `PanelShell.tsx`)
  - Слой `panel-cinema-bg` только в `PanelShell`; с `.root[data-color-preset="starWars"]` сняты fullscreen-анимации.

### Operational Notes

- Обновление **панели (`sharx`) и нод (`sharxnode`)** рекомендуется вместе: фикс старта ядер на воркере + push с панели.
- Если в логах ноды **`Config pull: panel HTTP 401: Неизвестный узел для адреса`** — проверьте **`NODE_ADDRESS`** в compose и адрес ноды в панели; при устаревшем `nodeId` в `node-config.json` после v1.5.0 воркер сбросит id и повторит pull по адресу.
- Ошибки **`127.0.0.1:62789 connection refused`** в первые секунды после старта — ожидаемая гонка до поднятия Xray API; не путать с полным отказом ядра.
- Тема Star Wars — только UI панели; на работу Xray/Telemt не влияет.

### 📦 Upgrade from v1.4.9

1. Бэкап `.env` и БД (миграций нет).
2. Обновите **`sharx`** и **`sharxnode`** до **`1.5.0`** / **`latest`**.
3. Перезапустите ноды; убедитесь, что **`PANEL_URL`** и **`NODE_ADDRESS`** в env совпадают с панелью.
4. Проверьте:
   - **Settings → тема панели:** Star Wars, фон в панели без лишних артефактов на `/panel/sub`.
   - **Clients:** выбор даты истечения → **00:00**.
   - **Inbounds → Xray config:** автодополнение порта, «Обновить из формы» не ломает JSON.
   - **Nodes:** после рестарта воркера Xray/Telemt поднимаются без ручного apply (или в логах `Worker config recover` / `Config pull: applied`).

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
