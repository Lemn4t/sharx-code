## Release v1.4.9-dev

**Release Date:** June 4, 2026

**Dev-канал** относительно **v1.4.8**: панель (inbounds tag/JSON-редактор, чекбоксы выбора), совместимость VLESS XTLS flow с транспортом, WireGuard peer email для статистики Xray, проверка портов на нодах при создании inbound.

**Миграция БД:** нет.

---

### Features & Improvements

- **Колонка и поле Xray tag в Inbounds** (`web/service/inbound_tag.go`, `web/controller/inbound.go`, `panel/lib/inboundTag.ts`, `panel/components/InboundsPage.tsx`)
  - Строковый tag (не только цифры): буквы/цифры, `.`, `_`, `:`, `-`, до 64 символов; зарезервирован `api`.
  - Пустое поле → автогенерация (`inbound-{port}`, `inbound-{listen}:{port}` или `inbound-{id}` в multi-node).
  - Уникальность в БД; при смене tag возможен полный перезапуск Xray (как при других структурных изменениях).

- **JSON-редактор inbound (Form | JSON | Core preview)** (`panel/components/inbounds/InboundJsonEditor.tsx`, `panel/lib/inboundJsonSanitize.ts`, `panel/lib/inboundJsonMonacoSchemas.ts`, `InboundsPage.tsx`)
  - Редактирование `streamSettings`, `sniffing`, серверных полей `settings` с Monaco и схемами.
  - Неподдерживаемые ключи отрезаются при Apply / Save (round-trip через form parsers).

- **Чекбоксы-карточки для массового выбора** (`panel/components/ui/checkbox-option.tsx`, `checkbox-field.tsx`)
  - **Groups**, **Inbounds** (ноды), **Xray core config profiles**, **Nodes** (radio профиля).

- **Multi-node: проверка порта до создания inbound и откат при ошибке назначения** (`web/controller/inbound.go`, `web/service/node.go`)
  - `CheckInboundPortConflictOnNodes` перед add.
  - При сбое `AssignInboundToNodes*` — удаление только что созданного inbound.

- **Nodes / Inbounds: режимы Table · List · Tiles с сохранением в localStorage** (`panel/lib/listViewModeStorage.ts`, `NodeListViews.tsx`, `InboundListViews.tsx`, `NodesPage.tsx`, `InboundsPage.tsx`)
  - Единый toolbar: переключатель вида + сворачиваемая панель фильтров над контентом (работает одинаково в таблице, списке и плитках).
  - List/Tiles повторяют действия таблицы (enable, delete; на нодах — Xray/Telemt, metrics).

### Bug Fixes

- **VLESS XTLS flow на несовместимом транспорте (XHTTP, WS и т.д.)** (`web/service/vless_flow_compat.go`, `sub/subService.go`, `web/service/client.go`, `panel/lib/xrayOutboundForm.ts`, `OutboundsBuilder.tsx`)
  - Flow очищается при сохранении inbound, не попадает в share links и payload пользователя Xray, если transport ≠ TCP+TLS/REALITY (и не VLESS Encryption).
  - В форме outbound flow disabled / сброс при переключении на XHTTP.

- **WireGuard: email на peer для статистики Xray** (`web/service/wireguard_inbound_settings.go`)
  - При санитизации settings: `email` из `name` / `clientName` / `clientEmail`; лишние panel-поля убираются из JSON для ядра.

### Operational Notes

- **Dev-образы:** `sharx:1.4.9-dev` / `latest-dev`, `sharxnode:1.4.9-dev` / `latest-dev` (см. Harbor).
- Обновление **только панели (`sharx`)** достаточно для UI и API inbound/tag/JSON; worker нужен, если используете проверку портов на нодах при создании inbound в multi-node.
- После обновления: пересоберите/обновите подписку у клиентов с VLESS+XHTTP — в ссылках не должно быть `flow=xtls-rprx-*`.

### 📦 Upgrade from v1.4.8

1. Бэкап `.env` и БД (миграций нет).
2. Обновите образ **`sharx`** до **`1.4.9-dev`** / **`latest-dev`** (и **`sharxnode`** при multi-node с новой проверкой портов).
3. Проверьте:
   - **Inbounds:** колонка **Tag**, редактирование tag в форме; JSON-вкладка Apply/Save.
   - **VLESS + XHTTP:** в подписке и в ядре нет `flow` на клиенте.
   - **WireGuard inbound:** трафик по peer учитывается в Xray stats (email на peer).
