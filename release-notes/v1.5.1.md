## Release v1.5.1

**Release Date:** June 6, 2026

Релиз **v1.5.1** относительно **v1.5.0**: исправлена доставка meta-параметров подписки в **Happ** (`Hide-Settings` и др.), выбор канала доставки для каждого параметра Response rules (как у Provider ID), фикс загрузки переводов панели.

**Миграция БД:** нет.

---

### Features & Improvements

- **Подписка: выбор способа доставки meta-параметров** (`panel/components/settings/subscription/ResponseRulesEditor.tsx`, `RoutingProfilesEditor.tsx`, `panel/lib/sharxSubpageConfig.ts`, `web/service/sharx_subpage_config.go`, `sub/subController.go`, `sub/happ_meta.go`)
  - Для **Profile title**, **update interval**, **announce**, **support URL**, **profile web page**, **extra parameters** и **routing** — выбор: **HTTP-заголовок**, **комментарий в теле** (`#key: value`), **оба** или **отключено**.
  - Модель как у **Provider ID** в настройках панели; старые конфиги без `*Delivery` по умолчанию отдают только HTTP-заголовок.

- **Happ: корректный Content-Type для meta-заголовков** (`sub/ua_dispatch.go`, `sub/subController.go`)
  - User-Agent **Happ** получает `Content-Type: application/json; charset=utf-8` — по [документации Happ](https://www.happ.su/happ/dev-docs/app-management) иначе `Hide-Settings`, `providerid` и другие meta-headers не применяются.
  - **v2rayTun** по-прежнему получает `text/plain`.

### Bug Fixes

- **Happ: Hide-Settings не скрывал настройки серверов** (`sub/subController.go`, `sub/happ_meta.go`)
  - Заголовок `Hide-Settings: 1` передавался, но игнорировался из‑за `Content-Type: text/plain`.
  - Дополнительно: при выборе **Body** / **Header + body** в конфигураторе в тело подписки добавляется `#hide-settings: 1` (запасной канал по документации Happ).

- **Панель не стартовала: ошибка TOML переводов** (`web/translation/translate.en_US.toml`, `web/translation/translate.ru_RU.toml`)
  - `extraHeaderDelivery` был и строкой, и таблицей — `toml: expected extraHeaderDelivery to be a table, not a value`.
  - Метка поля переименована в `deliveryLabel`; варианты вынесены в `[subBuilder.responseRules.extraHeaderDelivery]`.

### Operational Notes

- После обновления **пересохраните** конфиг страницы подписки и **обновите подписку** в Happ.
- Для **Hide-Settings** рекомендуется доставка **Header + body** и значение `1`.
- Обновляется только **панель (`sharx`)**; ноды (`sharxnode`) менять не обязательно, если не трогали multi-node.

### 📦 Upgrade from v1.5.0

1. Бэкап `.env` и БД (миграций нет).
2. Обновите **`sharx`** до **`1.5.1`** / **`latest`**.
3. Проверьте:
   - **Subscription builder → Response rules:** у параметров есть выбор доставки; для Hide-Settings — **Header + body**.
   - **Happ:** после обновления подписки настройки серверов скрыты (`Hide-Settings: 1`).
   - Панель стартует без ошибки TOML в логах.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
