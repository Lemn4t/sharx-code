## Release v1.4.6

**Release Date:** June 1, 2026

Релиз **v1.4.6** относительно **v1.4.5**: feature-релиз с совместимостью Xray 2026 (TLS pinning вместо `allowInsecure`), расширенными настройками IP-лимита на уровне панели, bulk-назначением клиентов в группы, исправлением подписки MTProto в multi-node и автообновлением активных geofile по source URL.

**Миграция БД:** `0044_client_blocked_session_ip_expires.sql` — колонка `expires_at` в `client_blocked_session_ips` для временных блокировок IP при auto IP-limit ban.

---

### Bug Fixes

- **На странице подписки показывался только первый MTProto proxy в multi-node** (`panel/components/sub/blocks/InstallationGuideBlock.tsx`, `AddToAppBlock.tsx`, `panel/components/sub/types.ts`)
  - **Причина:** UI брал `mtProtoLinks[0]` / один `tgProxyLink`, хотя бэкенд (`TelemtTgProxyLinesForSubscription`) уже отдавал все ссылки с разных нод.
  - **Следствие:** при нескольких telemt-инбаундах / нодах пользователь видел одну кнопку «Add proxy», остальные прокси были недоступны в installation guide.
  - **Фикс:** компонент `TelegramMtProtoProxyList` рендерит **все** ссылки с подписью (`tgProxyDisplayLabel`), copy/QR для каждой; `resolveMtProtoLinks` предпочитает `mtProtoLinks` из API с fallback на legacy parse из `links`.

- **Xray ≥ 2026 отклоняет клиентские outbounds с `allowInsecure` и устаревшими TLS-полями** (`xray/tls_sanitize.go`, `sub/subJsonService.go`, `sub/subService.go`, `panel/lib/inboundDefaults.ts`)
  - **Причина:** подписка JSON / share links / runtime template копировали inbound `tlsSettings` «как есть», включая deprecated `allowInsecure` и `pinnedPeerCertificateChainSha256`; Trojan outbounds могли получать лишний `flow`.
  - **Следствие:** после обновления Xray-core клиенты с self-signed / pinned TLS не подключались; конфиг мог не проходить валидацию.
  - **Фикс:** централизованный `SanitizeClientTLSSettings` — удаляет `allowInsecure`, нормализует pin в `pinnedPeerCertSha256`, поднимает `fingerprint` / `echConfigList` из legacy nested `settings`. Панель: UI-блок `InboundTlsCertPinBlock` для генерации self-signed cert + SHA256 pin; дефолты inbound TLS переведены на pinning вместо `allowInsecure`.

- **Сброс выбранных inbound'ов при возврате из редактора группы** (`panel/components/GroupsPage.tsx`)
  - **Причина:** draft inbound selection сбрасывался при каждом re-render / навигации «назад» из шага выбора инбаундов.
  - **Фикс:** `inboundDraftGroupIdRef` сохраняет draft только пока редактируется та же группа; при успешном save или смене группы draft очищается.

### Features & Improvements

- **Настройки IP-лимита на уровне панели** (`panel/components/SettingsPage.tsx`, `web/service/ip_limit_setting.go`, `web/job/check_client_ip_limit_job.go`, `web/service/client_session_block.go`)
  - **Settings → IP limit:**
    - `ipLimitGlobalEnable` — глобальный master-switch enforcement job'а (по умолчанию `true`).
    - `ipLimitCheckIntervalSec` — интервал проверки, 5–600 сек (throttle внутри job'а, tick `@every 1s`).
    - `ipLimitBanDurationSec` — длительность block-ban для лишних IP, 0 = permanent, до 30 дней.
    - `ipLimitEnforcement` — `drop` | `block` | `drop_and_block`.
    - `ipLimitExcessPolicy` — какие IP считать «лишними»: `newest` (по умолчанию) или `oldest` (ranking по `lastSeen` из online sessions).
  - **Block mode:** `BlockSessionIPInternal` пишет IP в `client_blocked_session_ips` с `expires_at`; routing/subscription hot-unblock при истечении (`ExpireDueSessionIPBlocks` в начале каждого цикла job'а).
  - Per-client `ipLimitEnabled` / `maxIPs` (v1.4.5) по-прежнему задают лимит; новые настройки управляют **как** панель реагирует на превышение.

- **Bulk assign clients to group** (`panel/components/ClientsPage.tsx`, `POST /panel/group/{id}/assignClients`)
  - В списке клиентов: multi-select → «Assign to group» → выбор группы → массовое назначение через существующий API `AssignClientsToGroup`.
  - Удобно для миграции клиентов между группами без поштучного редактирования карточек.

- **Автообновление активных geofile** (`panel/components/XrayPage.tsx`, `web/job/geofile_auto_update_job.go`, `web/service/geofile_asset.go`)
  - **Xray → Geo:** переключатель `geofileAutoUpdateEnable` + интервал 1–168 ч.
  - **Job** (`@every 1h`, throttle по интервалу): для каждого `geofile_assets` где `is_active = true` и `source_url <> ''` — скачать URL, сравнить SHA256; при изменении — `UploadGeofileAsset` + `ApplyGeofileAsset` (панель + worker nodes).
  - Ручные upload'ы без source URL **не** участвуют в auto-update.

- **TLS cert pin UI для inbound'ов** (`panel/components/inbounds/InboundTlsCertPinBlock.tsx`, `web/controller/inbound.go`)
  - Генерация self-signed cert с автозаполнением `pinnedPeerCertSha256`.
  - Вычисление pin из загруженного PEM (`generateSelfSignedTls` / compute pin endpoints).

### Operational Notes

- Обновление затрагивает в первую очередь **панель** (`sharx`). Образ **`sharxnode`** обновления **не требует** — geofile apply и IP-limit drop используют существующие node API.
- **Миграция 0044** применяется автоматически при старте панели. Бэкап БД перед обновлением — как обычно.
- **Xray 2026 / TLS:** после обновления пересохраните inbound'ы с self-signed TLS или проверьте, что в stream settings есть `pinnedPeerCertSha256`. Старый `allowInsecure: true` больше не попадает в subscription JSON.
- **IP limit:** если раньше использовался только drop — поведение по умолчанию `drop_and_block` + ban 3600 сек + kick newest. При необходимости верните `drop` only в Settings.
- **Geofile auto-update:** включайте только для активных файлов, загруженных **по URL** (source URL сохранён). Интервал по умолчанию 24 ч.

### 📦 Upgrade from v1.4.5

1. Бэкап `.env` и БД.
2. Обновите образ **`sharx`** до версии **`1.4.6`** / **`latest`**. **`sharxnode`** можно не трогать.
3. Дождитесь применения миграции `0044` (лог старта панели / `SELECT expires_at FROM client_blocked_session_ips LIMIT 1;`).
4. Проверьте:
   - **Settings → IP limit** — интервал, режим enforcement, ban duration сохраняются.
   - Клиент с `Limit concurrent IPs`: при превышении — drop и/или block согласно настройкам; временный block снимается после `banDuration`.
   - **Clients → bulk select → Assign to group** — клиенты попадают в выбранную группу.
   - **Subscription page** (multi-node telemt): все MTProto proxy links видны в installation guide.
   - **Xray → Geo:** auto-update toggle + interval; для active geofile с URL — job в логах `GeofileAutoUpdateJob: updated and applied N geofile asset(s)` при изменении upstream.
   - Inbound с TLS: subscription JSON содержит `pinnedPeerCertSha256`, без `allowInsecure`.
