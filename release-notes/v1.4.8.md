## Release v1.4.8

**Release Date:** June 1, 2026

Релиз **v1.4.8** относительно **v1.4.7**: patch-релиз с исправлением TLS certificate pinning в **URI share links** (text-подписка). JSON-подписка уже работала корректно с v1.4.6 через `SanitizeClientTLSSettings`.

**Миграция БД:** нет.

---

### Bug Fixes

- **TLS pin не попадал в URI share links при self-signed / pinned сертификатах** (`sub/share_link_tls.go`, `sub/subService.go`, `sub/host_subscription_overrides.go`)
  - **Причина:** генераторы `vless://`, `trojan://`, `ss://`, `vmess://` и `hysteria2://` копировали только legacy `allowInsecure` / `insecure=1`, но не `pinnedPeerCertSha256` из inbound TLS settings. Клиенты Xray ≥ 2026 и Hysteria2 с pinning не могли подключиться по ссылке из text-подписки, хотя JSON-подписка была корректной.
  - **Фикс:**
    - **VLESS / Trojan / Shadowsocks:** query param `pcs=<sha256>` (вместо `allowInsecure=1` при наличии pin).
    - **VMess:** поле `pinnedPeerCertSha256` в base64 JSON payload.
    - **Hysteria2:** `pcs` + `pinSHA256` (без `insecure=1` при pin).
    - **Host overrides:** `SubscriptionAllowInsecure` на Host не перезаписывает pin, если он уже задан inbound'ом.
    - **External proxy (forceTls=none):** pin-поля (`pcs`, `pinnedPeerCertSha256`, `insecure`, `pinSHA256`) корректно убираются вместе с остальными TLS-параметрами.

### Operational Notes

- Обновление **только панели (`sharx`)** достаточно — worker / sharxnode не затронут.
- Затронуты только **text share links** в подписке (`/sub/...` без JSON). JSON-подписка и runtime Xray config не менялись.
- После обновления перегенерируйте подписку в клиенте (refresh subscription), чтобы получить ссылки с `pcs` / `pinnedPeerCertSha256`.

### 📦 Upgrade from v1.4.7

1. Бэкап `.env` и БД (на всякий случай; миграций нет).
2. Обновите образ **`sharx`** до **`1.4.8`** / **`latest`**.
3. Проверьте inbound с self-signed TLS + certificate pin:
   - Text-подписка: `vless://` / `trojan://` / `ss://` содержат `pcs=…` (не `allowInsecure=1`).
   - `vmess://` (decoded JSON): поле `pinnedPeerCertSha256`.
   - `hysteria2://`: `pcs=…` и `pinSHA256=…`, без `insecure=1`.
   - Клиент подключается по ссылке из подписки без ручного `allowInsecure`.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
