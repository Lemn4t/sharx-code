## Release v1.5.2

**Release Date:** June 6, 2026

Релиз **v1.5.2** относительно **v1.5.1**: правка клиентов без лишнего `apply-config` на нодах (HWID, announce), фикс скролла модалки Inbounds при чекбоксах (Sockopt / TCP Fast Open).

**Миграция БД:** нет.

---

### Bug Fixes

- **Клиенты: HWID / announce больше не ребутят Xray на нодах** (`web/service/client.go`, `web/service/client_inbound_ids_test.go`)
  - Панель при каждом сохранении клиента отправляла `inboundIds` даже без изменений; бэкенд считал это сменой инбаундов и пересобирал Settings → `apply-config` на всех нодах.
  - Теперь сравниваются фактические назначения инбаундов; изменения только **HWID**, **announce**, comment, subId и др. metadata не трогают Xray.
  - Сброс трафика / срока / enable по-прежнему синхронизирует ядро.

- **Inbounds: скролл модалки при чекбоксах** (`panel/components/ui/checkbox-field.tsx`, `panel/components/ui/modal.tsx`)
  - Клик по **TCP Fast Open** (Sockopt) и другим чекбоксам в длинной форме сбрасывал прокрутку модалки наверх (фокус на `sr-only` input).
  - Клик по label больше не переводит фокус на скрытый input; клавиатурная навигация сохранена.

- **Сборка web/service** (`web/service/xray_hysteria_tls.go`)
  - `logger.Debug` с `%q` заменён на `Debugf` (ошибка компиляции vet).

### Operational Notes

- Обновляется **панель (`sharx`)**; **`sharxnode`** менять не обязательно.
- После обновления проверьте: правка announce/HWID у клиента **не** вызывает рестарт Xray в логах нод; в Inbounds → Transport → Sockopt галочка **TCP Fast Open** не сбрасывает скролл.

### 📦 Upgrade from v1.5.1

1. Бэкап `.env` и БД (миграций нет).
2. Обновите **`sharx`** до **`1.5.2`** / **`latest`**.
3. Проверьте:
   - **Clients:** изменение HWID или announce — без `apply-config` на нодах.
   - **Inbounds → VLESS → Transport → Sockopt:** TCP Fast Open — скролл модалки на месте.

```yaml
services:
  sharx:
    image: {{SHARX_TAG_LATEST}}
  sharxnode:
    image: {{SHARXNODE_TAG_LATEST}}
  postgres:
    image: {{HARBOR_HOST}}/{{HARBOR_PROJECT}}/postgres:latest
```
