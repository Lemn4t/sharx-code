# SharX v1.6.3 (dev)

## Fixes

### AmneziaWG sidecar — `device or resource busy`

When an inbound was re-synced quickly, the old `amneziawg-go` process could still hold the TUN interface name. The sidecar manager now:

- waits for the previous process to exit (with timeout + kill),
- runs `ip link delete <iface>` before starting a new instance,
- uses shorter interface names (`awg{inboundId}`) to match panel payloads.

### Subscription client `.conf` (WireGuard / AmneziaWG)

Client wg-quick blocks are built through one shared helper so WireGuard and AmneziaWG export the same `[Interface]` / `[Peer]` shape. AmneziaWG adds `MTU` and obfuscation keys (`Jc`, `H1`–`H4`, `S1`–`S4`) in `[Interface]`.

Server peer `AllowedIPs` in `BuildAmneziaWGServerConf` no longer falls back to `{publicKey}/32` (invalid); peers without `allowedIPs` are skipped.

**Node setconf format:** sidecar payloads now use `awg setconf`-compatible server config (no `Address`, `MTU`, or `PostUp`/`PostDown` — those are wg-quick-only). Tunnel address/subnet travel in payload fields; the manager runs `ip addr add` + iptables NAT after setconf.

**Docker `awg` binary:** image build now installs `linux-headers` + `libmnl-dev` and fails if `awg` is missing (previously setconf was silently skipped).

Server `.conf` preview still includes **PostUp/PostDown** for wg-quick; sidecar applies the same routing via `ensureTunnelRouting`.

Sidecar logs: `awg setconf OK` / failure output, warning when `awg` binary is missing.

**setconf retry:** sidecar waits for UAPI (up to ~3s) and retries `awg setconf`; previously a race left `amneziawg-go` running without peers/listen port until the conf hash changed.

**Multi-node UDP endpoint:** AmneziaWG / WireGuard subscriptions prefer node published addresses even when Host subscription mode is «replace» (Host CDN domains do not receive raw UDP).

**Traffic accounting:** node polls `awg show dump` and merges per-peer transfer deltas into `/stats` (client + inbound counters), same pattern as Telemt. Online: recent handshake (≤3 min) or traffic delta; multi-node and standalone panel merge into `onlineClients`. Server setconf no longer includes `PersistentKeepalive` on peers (avoids «no known endpoint» errors for idle clients).

**Node image:** runtime includes `iptables`; `sysctl ip_forward` warning in Docker is suppressed when the filesystem is read-only (enable forwarding on the host).

**iptables NAT (sidecar):** `iptables -t nat` must precede `-A`/`-C` (PR #14); rule existence check (`-C`) uses the same argument order so MASQUERADE is not re-added on every sync.

### Panel

Subscription metadata filter recognizes AmneziaWG obfuscation lines so AWG-only subscriptions still export QR / copy correctly.

### Panel — QR в «Ключах подключения»

**AmneziaVPN (публичная подписка):** все ключи `vless://` / `vmess://` / … **и** блоки AmneziaWG `.conf` (Download / Copy на каждый; для ссылок — Import / QR), как в [документации Amnezia](https://docs.amnezia.org/documentation/instructions/connect-via-config/). AWG-only: карточка AmneziaVPN показывает `.conf` для импорта в приложение.

**AmneziaWG (публичная подписка):** на шаге импорта — **Download .conf** вместо копирования URL подписки; QR по-прежнему из wg-quick блока.

**Nodes → добавление ноды:** шаблон docker compose в UI включает `devices: [/dev/net/tun:/dev/net/tun]` (рядом с `NET_ADMIN`).

Обновление **только бинарника/API без пересборки panel** не меняет QR: UI встроен в Docker-образ при `npm run build`. Нужен **`docker compose build sharx && docker compose up -d --force-recreate sharx`**.

API `GET /panel/client/links/:id` теперь отдаёт поле **`wgConf`** — чистый `[Interface]/[Peer]` для WireGuard и AmneziaWG (работает сразу после рестарта `sharx`, panel подхватит после rebuild).

**Clients → connection key QR:** AmneziaWG теперь кодирует только wg-quick `.conf` (раньше — весь panel-текст с `AmneziaWG (UDP) — …`, который приложение отклоняет).

## Upgrade

Replace panel + node images/binaries tagged `v1.6.3-dev`. No database migration beyond v1.6.2.

Docker node still requires `devices: [/dev/net/tun:/dev/net/tun]` and `NET_ADMIN` for userspace AmneziaWG.
