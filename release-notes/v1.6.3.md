# SharX v1.6.3 (dev)

## Fixes

### AmneziaWG sidecar — `device or resource busy`

When an inbound was re-synced quickly, the old `amneziawg-go` process could still hold the TUN interface name. The sidecar manager now:

- waits for the previous process to exit (with timeout + kill),
- runs `ip link delete <iface>` before starting a new instance,
- uses shorter interface names (`awg{inboundId}`) to match panel payloads.

### Subscription client `.conf` (WireGuard / AmneziaWG)

Client wg-quick blocks are built through one shared helper so WireGuard and AmneziaWG export the same `[Interface]` / `[Peer]` shape. AmneziaWG adds `MTU` and obfuscation keys (`Jc`, `H1`–`H4`, `S1`–`S4`) in `[Interface]`.

Server peer `AllowedIPs` in `BuildAmneziaWGServerConf` no longer falls back to `{publicKey}/32` (invalid); peers without `allowedIPs` are skipped.

### Panel

Subscription metadata filter recognizes AmneziaWG obfuscation lines so AWG-only subscriptions still export QR / copy correctly.

**Clients → connection key QR:** AmneziaWG now encodes only the `[Interface]` / `[Peer]` `.conf` block in the QR (same as WireGuard). Previously the QR included panel hint lines (`AmneziaWG (UDP) — …`, `Endpoint:`, etc.), which the AmneziaWG app rejects.

## Upgrade

Replace panel + node images/binaries tagged `v1.6.3-dev`. No database migration beyond v1.6.2.

Docker node still requires `devices: [/dev/net/tun:/dev/net/tun]` and `NET_ADMIN` for userspace AmneziaWG.
